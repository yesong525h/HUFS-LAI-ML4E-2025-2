# 1. 데이터 수집 및 구성 전략

 - 주요 데이터 출처 : 약 100페이지 분량의 강의 자료의 원본 PDF 및 PPT 파일 (예: `ml4e-lecture-week10`, `ml4e-lecture-week9`, `NLP_10`).
   - 수집 방법 :
    1.  원문 텍스트 추출 : 모든 강의 자료의 영어 텍스트를 문장 단위로 수동 추출하여 확보했습니다.
    2.  번역 데이터 확보 : API 인증 문제로 인해, 모든 원문 문장에 대한 수동 번역을 통해 machine_translation 컬럼을 확보했습니다.
   - 데이터 구조 :
     data/data.csv는 다음 5개 컬럼을 포함합니다. (document_id, page_num, source_text, machine_translation, needs_correction)

# 2. 데이터 분석 및 특징 파악 (EDA 결과)

## 2.1. 데이터 품질 문제 및 전처리 기록

- 용어 불일치 : 수동 분석(needs_correction) 결과, 전문 용어의 표준화 문제가 다수 발견되었습니다.
  주요 사례로 'Convolution'을 '컨볼루션' 대신 '합성곱'으로, 'element-wise'를 '성분별'로 수정해야 하는 오류가 기록되었습니다.
- 수식 처리 : 수학 수식(예: $\lambda_1, W_{hh}$)은 원형 그대로 보존하여 번역 오류를 방지했습니다.

## 2.2. 핵심 통계 및 오류 경향 분석 

| 통계 지표 | 값 |
| :--- | :--- |
| 전체 오류율 (Needs Correction=True) | 6.72% |
| 수정 필요 문장의 평균 길이 | 8.31 단어 |
| 수정 불필요 문장의 평균 길이 | 8.17 단어 |

- 주요 발견 사항 : 박스 플롯 시각화 결과, 수정 필요(`True`) 문장의 평균 길이가 수정 불필요(`False`) 문장의 평균 길이보다 약간 더 길다는 경향이 확인되었습니다. 
문장 길이의 차이(0.14단어)가 크지 않으므로, 이 결과는 단순히 문장 길이가 길다는 사실보다는 '복잡한 문장 내에 포함된 전문 용어와 문맥의 복잡성'이 오류를 유발하는 주된 원인임을 시사합니다.
