프로젝트 제안서: 칼로리 및 영양소 기반 다이어트 식단 추천 시스템

1.	프로젝트 개요
   
1.1. 프로젝트명

간단 영양소 분석 기반 식단 추천 도구

1.2. 프로젝트 설명

사용자가 하루 동안 섭취한 음식 정보를 입력하면, 칼로리·탄수화물·단백질·지방(탄단지) 기준으로 남은 하루 권장 영양 비율을 계산하여 다음 식사 추천을 제공하는 간단한 식단 조절 도구이다.
데이터는 한식·양식 등 특정 국가에 제한하지 않고 다양한 메뉴를 포함하지만, 모델은 복잡한 ML 대신 간단한 규칙 기반 혹은 소규모 추천 로직으로 구성한다.

2.	해결하고자 하는 문제
   
다이어트 중 “오늘 점심 많이 먹었는데 저녁은 뭐 먹지?”라는 고민이 반복됨
칼로리나 탄단지 계산이 귀찮아 식단 조절이 어려움
다양한 외식/배달 메뉴로 인해 하루 영양 균형 파악이 어려움
복잡한 모델 없이 현실적으로 구현 가능한 식단 추천 솔루션이 부족함
→ 입력된 식사 정보를 즉시 분석해 간단하게 다음 식사를 추천해주는 도구가 필요함.

3.	실제 사용 시나리오

사용자가 점심에 먹은 음식을 입력한다. -> 시스템이 칼로리·탄단지 정보를 계산한다. -> 남은 하루 권장 영양소를 계산한다. -> 저녁 식단을 2~3가지 형태로 간단하게 추천한다. 예: “지방이 과다 → 저녁은 저지방·고단백 추천”/“칼로리 이미 충분 → 가벼운 샐러드류 추천” -> 사용자는 추천된 메뉴 중 선택하여 식단을 조절할 수 있음

4.	제안하는 솔루션
   
음식 데이터셋(칼로리·탄단지 중심) 구축, 섭취 영양소 계산 엔진, 규칙 기반 추천 모델 - 탄단지 비율이 기준보다 벗어나면 자동으로 “보정 메뉴” 추천 예: 단백질 부족 → 닭가슴살/두부/달걀 기반 메뉴/지방 과다 → 저지방 메인 + 채소 조합

5.	주요 기능
   
음식 선택 → 칼로리·탄단지 자동 계산
하루 권장 섭취량 대비 부족/과다 확인
기본 식단 템플릿 제시

6.	ML 활용 계획
    
Assignment 4:
대표 음식 데이터셋(한식/양식 등 기본 영양소 정보) 구축, 권장 칼로리·탄단지 계산 공식 설정

Assignment 5:
간단한 추천 알고리즘 구현(규칙 기반 우선), 이후 가능하면 clustering으로 “비슷한 패턴의 사용자 → 비슷한 식단 추천” 시도

Assignment 6:
실제 사용 시나리오 테스트

7.	기대 효과
    
단순하면서도 실질적 도움이 되는 다이어트 식단 추천 가능
하루 식단의 과다·부족을 쉽게 확인할 수 있음
외식/배달 중심 식습관에도 대응 가능
